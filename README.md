# 🛰 Self-Bot — ربات مدیریت زندهٔ پروفایل تلگرام

یک سلف‌بات کامل و آمادهٔ بهره‌برداری که با کتابخانهٔ **[gotd/td](https://github.com/gotd/td)** و سشن کاربر،
**اسم / نام خانوادگی / بایو** حساب تلگرام شما را به‌صورت **زنده و هوشمند** بر اساس متغیرها تغییر می‌دهد.
تمام مدیریت از طریق یک **ربات تلگرامی رابط** (Bot API + Webhook) انجام می‌شود — بدون نیاز به کار با خط فرمان.

> A complete, production-ready Telegram self-bot that live-updates your profile
> (first name / last name / bio) from configurable variables, fully managed
> through an interface bot. SQLite storage, no CGO, deploys locally or on Railway.

---

## ✨ امکانات

- 🔐 **ورود امن**: اشتراک‌گذاری شماره یا تایپ دستی → دریافت کد → رمز دومرحله‌ای (در صورت نبودن، خودکار رد می‌شود).
- 🎛 **سه فیلد پروفایل**: نام، نام خانوادگی، بایو — هرکدام با الگوی مستقل، فونت و حداقل فاصلهٔ بروزرسانی.
- 🧩 **متغیرها**: ترکیب آزاد متن ثابت و متغیر داخل الگو با نشانهٔ `{نام}`.
- ⏱ **بازهٔ مستقل برای هر متغیر**: دو متغیر زمانی در یک فیلد می‌توانند با سرعت‌های متفاوت تغییر کنند.
- 🔤 **۱۳ فونت یونیکد**: ضخیم، ایتالیک، اسکریپت، فراکتور، مونو، توخالی، دایره‌ای، اسمال‌کپس و… .
- 🌐 **دوزبانه** (فارسی/انگلیسی) با کلید ترجمه — قابل تغییر در لحظه.
- 📦 **خروجی/ورودی** کل پیکربندی به‌صورت فایل **JSON** و **TXT**.
- 🛡 **ضدبن**: میدلورهای `floodwait` و `ratelimit`، حداقل فاصلهٔ سراسری بین بروزرسانی‌ها، اثرانگشت دستگاهِ واقعی.
- 🚀 **دیپلوی دوگانه**: لوکال (long polling) یا Railway (webhook) — فقط با تنظیم `PUBLIC_URL`.

## ⚙️ پیش‌نیازها

1. **ربات رابط**: از [@BotFather](https://t.me/BotFather) یک ربات بسازید و `BOT_TOKEN` بگیرید.
2. **اعتبارنامهٔ MTProto**: از [my.telegram.org](https://my.telegram.org) مقادیر `API_ID` و `API_HASH` را بگیرید.
3. **شناسهٔ عددی خودتان**: از [@userinfobot](https://t.me/userinfobot) بگیرید و در `WHITELIST` بگذارید.

## 🖥 اجرای لوکال

```bash
cp .env.example .env      # مقادیر را پر کنید (PUBLIC_URL را خالی بگذارید)
go run ./cmd/selfbot
```

در حالت لوکال `PUBLIC_URL` خالی است و ربات با **long polling** کار می‌کند.
سپس در تلگرام به ربات رابط `/start` بدهید.

---

## ☁️ دیپلوی روی Railway

1. این مخزن را به Railway متصل کنید (Deploy from GitHub repo). فایل `Dockerfile` و `railway.json` آماده‌اند.
2. در بخش **Variables** مقادیر `.env` را وارد کنید. حتماً `PUBLIC_URL` را برابر دامنهٔ عمومی سرویس بگذارید:
   `https://<your-app>.up.railway.app`  → وب‌هوک به‌صورت **خودکار** ثبت می‌شود.
3. یک **Volume** بسازید و روی مسیر `/data` متصل کنید، سپس `DB_PATH=/data/selfbot.sqlite` تنظیم کنید
   تا سشن و تنظیمات بعد از ری‌استارت باقی بمانند.
4. `PORT` را Railway خودش تزریق می‌کند؛ نیازی به تنظیم دستی نیست.

> در حالت وب‌هوک یک endpoint سلامت روی `/` برمی‌گرداند تا health-check سبز شود.

## 📋 نحوهٔ استفاده

1. `/start` → منوی اصلی.
2. **🔐 حساب کاربری → ورود** → شماره را به اشتراک بگذارید/تایپ کنید → کد → (در صورت وجود) رمز دومرحله‌ای.
3. **🧩 متغیرها → افزودن** → نوع را انتخاب، نام بدهید، تنظیمات را کامل کنید.
4. **🎛 فیلدهای پروفایل** → یک فیلد را باز کنید، الگو را با `{نام}` بسازید، فونت و فاصله را تنظیم و فعالش کنید.
5. **⏱ بروزرسانی زنده → روشن** (یا «بروزرسانی فوری»). `Tick` بازهٔ بررسی کلی است.
6. **📦 خروجی/ورودی** برای پشتیبان‌گیری یا انتقال تنظیمات (JSON/TXT).

---

## 🛡 نکات ضدبن

- بروزرسانی **بیش از حد مکرر** اسم پروفایل ریسک محدودیت دارد؛ بازه‌های کوتاه را محتاطانه انتخاب کنید.
- `MIN_UPDATE_GAP_SEC` یک کف امنیتی سراسری بین هر دو فراخوانی `updateProfile` است.
- ربات فقط وقتی پروفایل را تغییر می‌دهد که **مقدار واقعاً عوض شده باشد** (از آپدیت بی‌مورد پرهیز می‌شود).
- خطاهای `FLOOD_WAIT` به‌صورت خودکار مدیریت می‌شوند.

## 🗂 ساختار پروژه

```
cmd/selfbot/            entrypoint و سیم‌کشی
internal/config/        بارگذاری و اعتبارسنجی env
internal/storage/       SQLite (modernc, بدون CGO) + سشن gotd
internal/userbot/       کلاینت gotd/td، ورود تعاملی، آپدیت پروفایل
internal/variables/     فونت‌ها، متغیرهای آماده، موتور رندر
internal/scheduler/     حلقهٔ بروزرسانی زنده با بازهٔ مستقل هر متغیر
internal/botui/         ربات رابط: منوها، استیت‌ماشین، خروجی/ورودی
internal/i18n/          ترجمهٔ فارسی/انگلیسی
```
